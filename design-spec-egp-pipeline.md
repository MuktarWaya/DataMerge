# สเปกดีไซน์ (Design Specification) — สไตล์ "e-GP Pipeline"

สไตล์โดยรวม: **Neo-Brutalism แบบการ์ตูนอบอุ่น (Warm Cartoon Neubrutalism)**
เอกลักษณ์หลักคือ พื้นหลังสีครีม + เส้นขอบดำหนา + เงาแข็งสีดำไม่เบลอ (hard shadow) + มุมโค้งมนขนาดใหญ่ + สีสดใช้เป็นจุดเน้นทีละจุด ให้ความรู้สึกเป็นมิตร อ่านง่าย เหมือนบอร์ดเกม/สมุดโน้ต แต่ยังดูเป็นระบบงานจริงจังได้

---

## 1. แนวคิดหลัก (Design Principles)

1. **ทุกอย่างเป็น "การ์ด"** — เนื้อหาทุกก้อนอยู่ในกล่องขอบดำหนา มุมโค้ง มีเงาดำทึบเยื้องลงขวา
2. **เงาแข็ง ไม่เบลอ** — box-shadow เป็นสีดำล้วน offset ประมาณ 3–5px ไม่มีค่า blur เลย (นี่คือหัวใจของ Neo-Brutalism)
3. **สีพื้นเป็นกลาง สีสดเป็นสัญญะ** — พื้นครีม/เบจเป็นหลัก แล้วใช้สีสด (เหลือง เขียว แดง ฟ้า ม่วง) บอก "สถานะ" หรือ "การกระทำ" เท่านั้น เช่น เหลือง = แท็บที่เลือกอยู่, เขียว = ปุ่มหลัก/สำเร็จ, แดง = หยุด/อันตราย
4. **ตัวหนังสือหนา อ่านง่าย** — หัวข้อใช้น้ำหนัก Bold–ExtraBold สีดำสนิท ไม่ใช้สีเทาจาง
5. **ทุกปุ่มดูกดได้จริง** — ด้วยขอบ + เงา ทำให้ปุ่มดูนูน เวลากดให้ปุ่ม "ยุบ" (เงาหาย + เลื่อนลงตามระยะเงา)

## 2. ชุดสี (Color Palette)

### สีพื้นและโครงสร้าง
| ชื่อ | ค่าโดยประมาณ | ใช้กับ |
|---|---|---|
| Background (ครีม) | `#F6EFDC` | พื้นหลังทั้งหน้า มีลายจุด (dot grid) จางๆ |
| Surface (ขาวนวล) | `#FFFDF5` | พื้นการ์ด, ปุ่มปกติ, ช่องกรอกข้อมูล |
| Ink (ดำ) | `#1A1A1A` | เส้นขอบทุกเส้น, ตัวหนังสือหลัก, เงา |
| Track (เบจเข้ม) | `#EFE3C8` | รางของ progress bar, พื้นส่วนที่ยังไม่เต็ม |

### สีเน้น (Accent)
| ชื่อ | ค่าโดยประมาณ | ความหมาย / ใช้กับ |
|---|---|---|
| Yellow | `#FFD43B` | แท็บ/ปุ่มที่กำลังเลือกอยู่ (active state), ไฮไลต์ |
| Green | `#2F9E63` | ปุ่ม action หลัก ("ดึงข้อมูล", "ดึงมาทำต่อ"), จุดสถานะสำเร็จ |
| Green อ่อน | `#C9F0D8` | ป้ายสถานะเชิงบวก เช่น "แจ้งเตือน: เปิดอยู่" |
| Red | `#F0544F` | ปุ่มหยุด/ลบ, ไอคอนหมวดสำคัญ |
| Blue | `#4D96FF` | แถบกราฟจำนวน, ไอคอนขั้นตอน, ลิงก์/เลข PO |
| Purple | `#9B7BF0` | แถบกราฟเวลา, ไอคอนสรุปสถิติ |
| Cream Warning | `#FFF3C4` | กล่องโน้ต/หมายเหตุ, ปุ่มรอง ("แก้โน้ต"), แถบสถานะกำลังทำงาน |
| Chip เขียวอ่อน / ฟ้าอ่อน | `#DFF3E3` / `#DCE9FB` | ป้ายวันที่, ป้ายผู้รับผิดชอบ (เช่น "ส่วนกลาง") |

กติกาการใช้สี: **หนึ่งการ์ดควรมีสีสดเด่นแค่ 1–2 สี** ที่เหลือเป็นครีม/ขาว/ดำ เพื่อไม่ให้ลายตา

## 3. ตัวอักษร (Typography)

- **ฟอนต์แนะนำ (ไทย + อังกฤษ):** `IBM Plex Sans Thai`, `Noto Sans Thai`, หรือ `Anuphan` — เลือกตระกูลที่มีน้ำหนัก Bold/ExtraBold ชัดเจน ปลายมนเล็กน้อยจะเข้ากับสไตล์การ์ตูน
- **ฟอนต์ตัวเลข/โค้ด:** monospace เช่น `IBM Plex Mono` ใช้กับชื่อไฟล์, ชื่อตาราง, REF code (เห็นในหน้าบอร์ด เช่น `SQLite (egp_log.db)`)

| ระดับ | ขนาดโดยประมาณ | น้ำหนัก | หมายเหตุ |
|---|---|---|---|
| ชื่อหน้า (Page title) | 22–24px | 800 | มีไอคอนวงกลมนำหน้าเสมอ |
| หัวการ์ด (Card title) | 18–20px | 700–800 | คู่กับไอคอนสี่เหลี่ยมสี |
| เนื้อหา (Body) | 15–16px | 500–600 | สีดำ ไม่ใช้เทาอ่อน |
| ป้าย/ชิป (Label, chip) | 12–13px | 600 | ตัวเล็กแต่หนา |
| ตัวเลขค่า (Value) | 15–16px | 700 | ชิดขวาของแถบกราฟ |

## 4. เส้นขอบ เงา และมุมโค้ง (Border / Shadow / Radius)

นี่คือ "ลายเซ็น" ของดีไซน์นี้ ต้องคุมให้เท่ากันทั้งแอป

- **เส้นขอบ:** `2px solid #1A1A1A` สำหรับองค์ประกอบเล็ก, `2.5–3px` สำหรับการ์ดใหญ่
- **เงา:** `box-shadow: 4px 4px 0 #1A1A1A` (ไม่มี blur, ไม่มี spread) — องค์ประกอบเล็กใช้ `3px 3px 0`, การ์ดใหญ่ใช้ `5px 5px 0`
- **มุมโค้ง:**
  - การ์ดใหญ่: `20–24px`
  - ปุ่ม/แท็บ: ทรงแคปซูลเต็ม (`border-radius: 999px`)
  - ช่องกรอกข้อมูล: `14–16px`
  - ไอคอนสี่เหลี่ยม (icon tile): `14px` และ**เอียงเล็กน้อย** `transform: rotate(-4deg)` ให้ดูขี้เล่น
- **สถานะกด (pressed):** เลื่อนปุ่มลง-ขวาเท่าระยะเงา แล้วลดเงาเป็น 0 → `transform: translate(4px, 4px); box-shadow: none;`

## 5. พื้นหลัง (Background Texture)

พื้นครีมทั้งหน้า มี **ลายจุดตาราง (dot grid)** สีน้ำตาลอ่อนจางๆ ระยะห่างประมาณ 22–26px:

```css
body {
  background-color: #F6EFDC;
  background-image: radial-gradient(#D9CBA8 1.2px, transparent 1.2px);
  background-size: 24px 24px;
}
```

## 6. คอมโพเนนต์หลัก (Component Library)

### 6.1 แถบนำทางด้านบน (Top Tab Navigation)
- แถวปุ่มทรงแคปซูล เลื่อนแนวนอนได้ (horizontal scroll) มีแถบ scrollbar บางๆ ใต้แถว
- ทุกแท็บ: พื้นขาว ขอบดำ 2px เงา 3px ไอคอน + ข้อความหนา
- **แท็บที่เลือกอยู่:** เปลี่ยนพื้นเป็นเหลือง `#FFD43B` (อย่างอื่นเหมือนเดิม)
- เมนู 5 แท็บ: หน้าแรก / รันงาน / บอร์ด / สรุป / รายงาน — แต่ละแท็บมีไอคอนเส้น (line icon) ของตัวเอง
- เหนือแถบแท็บมี **ชื่อหน้า**: ไอคอนวงกลมดำ (ข้างในเป็นสัญลักษณ์สีเหลือง) + ชื่อแอป — ชื่อหน้าปัจจุบัน เช่น "e-GP Pipeline — หน้ารันงาน"

### 6.2 การ์ดเมนูหลัก (หน้าแรก)
- การ์ดเต็มความกว้าง เรียงแนวตั้ง ห่างกัน ~16–20px
- โครงสร้างภายใน: **ไอคอนสี่เหลี่ยมสี (56×56px, เอียง -4°)** → ชื่อเมนูสีเขียวเข้มหนา → คำอธิบายสั้น 1 บรรทัดสีดำ
- สีไอคอนแยกตามหมวด: เขียว (รันงาน), ฟ้า (บอร์ด), ม่วง (สรุป), แดง (รายงาน), ส้มครีม (Log)
- ทั้งการ์ดกดได้ (เป็นปุ่มใหญ่)

### 6.3 ปุ่ม (Buttons)
| ชนิด | หน้าตา | ตัวอย่าง |
|---|---|---|
| Primary | พื้นเขียว `#2F9E63` ตัวหนังสือขาวหนา เต็มความกว้าง มีไอคอน ↓ นำหน้า | "↓ ดึงข้อมูล", "↓ ดึงมาทำต่อ" |
| Danger | พื้นแดง แคปซูล ตัวขาว มีไอคอน ■ | "■ หยุด" |
| Secondary | พื้นครีมเหลืองอ่อน `#FFF3C4` ตัวดำ | "แก้โน้ต" |
| Neutral | พื้นขาว ตัวดำ | "รีเฟรช", "ออกจากระบบ" |
| Toggle/segment | ปุ่มติดกัน แคปซูลรวม ฝั่ง active เป็นเหลือง | "รายการ / บอร์ด" |

ทุกปุ่มมีขอบดำ + เงาแข็งเหมือนกันหมด

### 6.4 ช่องกรอกข้อมูล (Input / Search)
- พื้นขาว ขอบดำ 2px มุมโค้ง 14–16px **ไม่มีเงา** (ให้ต่างจากปุ่ม)
- Label อยู่เหนือช่อง ตัวหนา สีดำ พร้อมชื่อตัวแปรในวงเล็บ เช่น "เลข PO เริ่มต้น (start_po)"
- Placeholder เป็นตัวอย่างจริง สีเทา เช่น "เช่น 681799"
- ช่องค้นหามีไอคอนแว่นขยายด้านซ้ายในช่อง
- Dropdown: หน้าตาเหมือน input แต่มีไอคอนนำหน้า + ลูกศร ˅ ท้ายช่อง และแสดงจำนวนในวงเล็บ เช่น "ทุกบริษัท (62)"

### 6.5 แถบกราฟแนวนอน (Horizontal Bar / Funnel Chart) — หน้าสรุป
- แต่ละแถว: **ชื่อขั้นตอน (ซ้าย, ตัดคำด้วย …) | แถบกราฟ (กลาง) | ค่าตัวเลข (ขวา, ตัวหนา)**
- ตัวแถบ: ทรงแคปซูล ขอบดำ 2px / รางพื้นเบจ `#EFE3C8` / ส่วนเติมเป็นสีทึบชนขอบซ้าย
- ใช้สีแยกความหมายของกราฟ: **ฟ้า = จำนวน (ใบ)**, **ม่วง = เวลาเฉลี่ย**
- ค่า 0 หรือไม่มีข้อมูล: แถบว่างเหลือสีเติมนิดเดียวติดขอบซ้าย ค่าเป็น "0 ใบ" หรือ "—"
- หัวการ์ดมีไอคอนสี่เหลี่ยมแดงพร้อมสัญลักษณ์ + เส้นประคั่นใต้หัวข้อ

### 6.6 การ์ดสถานะกำลังทำงาน (Live Worker Card)
- การ์ดพื้น**ครีมส้มอ่อน** `#FDE8C8` มีขอบซ้ายหนาสีส้ม (accent border-left)
- ภายใน: จุดกลมเขียว (สถานะออนไลน์) + ชื่อผู้ใช้ + ป้ายดำแคปซูลบอกขั้นตอนปัจจุบัน (ตัวขาว) + ตัวเลข "0/1 โครงการ" ขวาสุด
- ปุ่ม "■ หยุด" สีแดง + แถบ progress ทรงแคปซูล + บรรทัด log ล่าสุด (มีจุด ••• กะพริบสีส้มนำหน้า)
- ด้านบนการ์ดมีภาพประกอบสไตล์ pixel art เล็กๆ + ข้อความ "1 กำลังทำงาน"

### 6.7 แถบสถานะย่อ (Status Strip)
- แคปซูลยาวเต็มความกว้าง พื้นครีม ขอบดำ ใช้แสดงสถานะสั้นๆ บนสุดของหน้า เช่น "worker ว่าง · ✅ ล่าสุด: ขั้นตอน 4: …"

### 6.8 Stepper / แท็บขั้นตอน (1-2-3)
- แถวปุ่มติดกันในกรอบเดียว แบ่ง 3 ช่องด้วยเส้นดำ
- แต่ละช่อง: วงกลมเลขมีสี (แดง/ฟ้า/ม่วง ขอบดำ) + ชื่อขั้น
- ช่องที่เลือกอยู่พื้นเหลืองอ่อน

### 6.9 บอร์ดคัมบัง (Kanban Board)
- คอลัมน์ตามขั้นตอน เลื่อนแนวนอน คอลัมน์กว้าง ~85% ของจอ (มองเห็นคอลัมน์ถัดไปโผล่ขอบ)
- หัวคอลัมน์: "④ ขั้น 2" + ป้ายดำแคปซูลตัวขาวบอกจำนวน (148)
- ใต้หัวมีปุ่มเขียวเต็มกว้าง "↓ ดึงมาทำ 148"
- **การ์ดงาน (Task card)** ประกอบด้วย:
  - แถวบน: checkbox สี่เหลี่ยมขอบดำ + ไอคอนจุดลาก (⠿) + **เลข PO สีฟ้า ตัวหนา**
  - ชื่อยา/รายการ (ตัวหนา ดำ)
  - ชิปวันที่ (พื้นเขียวอ่อน มีไอคอนปฏิทิน) เช่น "29/04/2569"
  - บรรทัดรหัส: "EGP 618581 · REF 69069507780"
  - **กล่องโน้ต** (ถ้ามี): พื้นเหลืองอ่อน ขอบซ้ายหนาเหลืองเข้ม ไอคอน 📝 เช่น "no Tax_no — 1940100082068"
  - ชิปผู้รับผิดชอบ: แคปซูลฟ้าอ่อน ไอคอนคน เช่น "ส่วนกลาง"
  - **แถวจุดความคืบหน้า 10 จุด**: จุดที่ผ่านแล้วเป็นเขียวทึบ ที่เหลือเป็นวงกลมโปร่งขอบดำ
  - แถวปุ่มล่าง: "แก้โน้ต" (ครีมเหลือง) + "↓ ดึงมาทำต่อ" (เขียว) แบ่งครึ่ง
- เหนือบอร์ด: แถบสรุป "Basla เปลี่ยนบัญชี · รวม 178 โครงการ • เสร็จครบ 0", toggle รายการ/บอร์ด, ปุ่มรีเฟรช, ช่องค้นหา, dropdown กรอง 2 ชั้น (บริษัท / วันที่)
- บรรทัดบอกแหล่งข้อมูลใช้ฟอนต์ mono: "ไฟล์งาน: ฐานข้อมูล `SQLite (egp_log.db)` · ตาราง `po_items`"

### 6.10 ตารางข้อมูล (Data Table)
- หัวตารางตัวหนา ไม่มีพื้นสี มีเส้นดำหนาคั่นใต้หัว
- เลื่อนแนวนอนได้ มี scrollbar บาง
- ค่าสำคัญ (เช่น EGP_CODE) เป็นตัวหนา
- เหนือมีข้อความสรุป: "ทั้งหมด 1 รายการ — กด "ดู" เพื่อดูรายละเอียดทั้งหมด"

### 6.11 ชิป/ป้าย (Chips & Badges)
- ทรงแคปซูล ขอบดำบาง 1.5–2px ตัวหนังสือเล็กหนา มีไอคอนอีโมจิ/เส้นนำหน้า
- โทนตามความหมาย: เขียวอ่อน (วันที่/เปิดใช้งาน), ฟ้าอ่อน (บุคคล), ดำ (จำนวน/สถานะระบบ ตัวขาว), เหลืองอ่อน (หมายเหตุ)

## 7. เลย์เอาต์และระยะห่าง (Layout & Spacing)

- **Mobile-first คอลัมน์เดียว** ความกว้างเนื้อหาเต็มจอ มี padding ซ้ายขวา ~14–16px
- ระยะห่างฐาน (spacing scale): 4 / 8 / 12 / 16 / 20 / 24px
- ระยะห่างระหว่างการ์ดใหญ่: 18–24px
- padding ภายในการ์ด: 18–22px
- เส้นคั่นภายในการ์ด: **เส้นประ** (`border-top: 2px dashed #D9CBA8`) ไม่ใช้เส้นทึบ
- ทุกอย่างที่ล้นจอใช้เลื่อนแนวนอน (แท็บ, บอร์ด, ตาราง) ไม่หดจนอ่านไม่ออก

  --chip-green: #DFF3E3;

## 9. โมชันและอินเทอร์แอกชัน (Motion)

- ใช้น้อยแต่ชัด: ปุ่ม "ยุบ" ตอนกด (0.08s), การ์ดขยับขึ้นเล็กน้อยตอน hover (desktop)

## 10. เช็กลิสต์คุมโทน (Do & Don't)

**Do**
- ขอบดำ + เงาแข็ง ให้เท่ากันทุกองค์ประกอบ
- สีสดใช้บอกความหมายเสมอ ไม่ใช้เพื่อสวยเฉยๆ
- ตัวหนังสือดำสนิท น้ำหนักหนา
- ไอคอนหัวการ์ดเอียงเล็กน้อยเพื่อความขี้เล่น

**Don't**
- ห้ามใช้เงาเบลอ/เงาเทาแบบ Material Design
- ห้ามใช้ gradient หรือ glassmorphism
- ห้ามใช้ตัวหนังสือสีเทาอ่อนกับเนื้อหาหลัก
- ห้ามใส่สีสดเกิน 2 สีในการ์ดเดียว
